# Handoff 格式规范 (Handoff Format)

## 工单流转状态
```
准备开发 → 准备审查 → 准备测试 → 完成
```

## Handoff 文件格式
```markdown
# Handoff: [A] → [B]

## 任务
[一句话描述要做什么]

## 需要看的文件
- [文件路径 1] — [为什么需要看]
- [文件路径 2] — [为什么需要看]

## 不许碰的文件
- [文件路径] — [为什么不能碰]

## 执行步骤
1. [步骤描述] → 验证: [验证方法]
2. [步骤描述] → 验证: [验证方法]

## 已完成的
- [已完成的事项]

## 产出物
- [文件路径] — [简要说明]

## 遗留问题
- [如有]

## 相关决策
- [ADR-XXX 或相关决定的简要描述]
```

## 五字段派单规则
主管（Coordinator）派单给子 agent 时，必须提供以下五个字段：
1. **任务描述** — 一句话说清楚做什么
2. **输入范围** — 只列需要看的文件（不多给信息）
3. **禁止范围** — 明确不许碰什么（防止越界）
4. **产出预期** — 期望交付什么、放在哪里
5. **决策上下文** — 之前做过什么相关决定（避免推翻旧决策）
