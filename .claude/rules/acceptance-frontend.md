# 前端验收标准 (Frontend Acceptance)

## 测试方式
- 使用 Playwright（浏览器自动化工具）实际操作
- 不靠"目测"、"理论上应该显示"判断

## 必测清单
- [ ] 桌面端（1920×1080）完整走通
- [ ] 手机端（375×667）完整走通
- [ ] 每个按钮点一遍，确认有反馈
- [ ] 每个输入框填一遍，确认能提交
- [ ] 空状态展示正确（无数据时不崩）
- [ ] 加载状态展示正确（有 spinner/skeleton）
- [ ] 错误状态展示正确（接口报错时有提示）
- [ ] 表单验证生效（必填项为空时阻止提交）
- [ ] 数据确实保存了（刷新页面后数据还在）
- [ ] 键盘操作可达（Tab/Enter/Escape）

## 不只是"看着对"
- 前端验收 = 视觉正确 + 交互正确 + 数据正确
- "页面渲染正常"只是第一步
- 必须验证：操作 → 接口调用 → 数据变更 → 页面更新 这条链路完整

## 设计标准
- 有独特的视觉风格，不是"AI 通用风格"
- 字体、间距、颜色一致
- 浅色/深色模式都可用（如果项目支持）
