# 决策记录节点 (Decision Nodes)

## 7 个必须记录的决策节点

以下每个节点发生时，必须在 DECISIONS.md 中新增一条 ADR：

### 1. 方案选定
- 触发：头脑风暴阶段选定了实现方案
- 记录：选了什么方案、排除了哪些备选方案、核心权衡点

### 2. 否决用户方案
- 触发：不同意用户提出的方案
- 记录：用户原方案是什么、为什么不可行、必须附上客观证据（不是主观意见）

### 3. Bug 修复策略
- 触发：确定了特定 bug 的修复策略
- 记录：bug 根因是什么、为什么用这个修法而不用其他修法

### 4. 工具/库选择
- 触发：选了某个技术工具或第三方库
- 记录：为什么用它而不用竞品、许可证兼容性、维护活跃度

### 5. 架构变更
- 触发：改变了系统架构设计
- 记录：旧架构 → 新架构的完整对比、变更原因

### 6. 已有方案发现
- 触发：发现某个问题已经被解决过
- 记录：发现时间、已有方案是什么、是否可以复用

### 7. 决定反转
- 触发：推翻了之前的一个决定
- 记录：原 ADR 编号、新旧方案完整对比、反转原因、影响范围

## 格式要求
每条 ADR 必须包含：
- 日期
- 状态（提议/已采纳/已废弃/已取代）
- 决策内容
- 备选方案
- 权衡分析
- 后果（正面和负面）
