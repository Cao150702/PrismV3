# 研究路由规则 (Research Routing)

## 查什么 → 去哪查

| 要查什么 | 去哪查 | 为什么 |
|----------|--------|--------|
| 某个库的 API 怎么用 | Context7 / 官方文档 | 训练数据可能过时，API 可能已变化 |
| 最新的最佳实践 | Exa（AI 语义搜索）/ Web 搜索 | 技术趋势变化快，旧建议可能已过时 |
| 同类产品怎么做的 | Web 搜索 + 深度阅读 | 避免闭门造车，参考业界做法 |
| 团队之前的经验 | mem0 / DECISIONS.md | 不要重复犯已经踩过的坑 |
| 项目内的代码/配置 | Grep + Read 工具 | 直接查代码库最准确 |

## 查询优先级
1. 先查项目内（DECISIONS.md → 代码搜索 → 记忆系统）
2. 再查官方来源（官方文档 → Context7）
3. 最后查社区（Web 搜索 → Exa → 技术博客）

## 禁止行为
- 禁止凭"记忆"写涉及支付、加密、认证的代码
- 禁止使用不熟悉的库而不查文档
- 禁止在不确定时"猜一个参数名"
- 查了文档发现和需求冲突 → 写 blocker，不要强行用

## 查询记录
当需要查外部资料时，在回复中标注：
- 查了什么来源
- 找到了什么信息
- 基于什么信息做了决定
