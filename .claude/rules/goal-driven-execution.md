# 目标驱动执行 (Goal-Driven Execution)

## 核心原则
"修好了"不是目标，"这个测试用例通过"才是。

## 执行规范

### 每步一个可验证目标
- 动手前先定义"做完的标准是什么"
- 目标必须可验证（能跑、能测、能看到结果）
- 格式：step → verify

示例：
```
Step: 创建用户注册接口
Verify: curl POST /register 返回 201 + 用户 ID

Step: 添加邮箱重复检查
Verify: 用已有邮箱注册，返回 409 Conflict
```

### 验证优先
- 先定义验证方法，再写实现代码
- 验证方法就是你的"考卷"
- 如果写不出验证方法，说明还没想清楚要做成什么样

### 完成后自检
- [ ] 最初的验证方法跑过了吗？
- [ ] 有没有引入新的问题？
- [ ] 用户能直接用吗？还是需要额外配置？
