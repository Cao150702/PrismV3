# 防摸鱼规则 (Anti-Slacking)

## 核心原则
AI 常见的敷衍行为：声称完成了但实际是半成品，或者在同一个错误上无限循环。

## 禁止行为

### 1. 声称"测试通过了"不附证据
- 必须贴出实际测试输出（终端日志、截图、Playwright 报告）
- "应该没问题" ≠ 验证通过
- 没有证据 = 没做

### 2. 代码里留 TODO 或占位符
- 禁止 `// TODO: 以后实现`
- 禁止 `// 待完善`
- 禁止 `pass  # 后续补充`
- 要么实现完整，要么明确告知用户"这一步需要你提供 X"

### 3. 声称"探索了但没发现"
- 必须展示搜索过程：用了什么关键词、查了哪些文件、为什么没找到
- "没找到"后面必须跟"接下来建议怎么做"

### 4. 无限重试
- 同一个修复尝试超过 3 次失败 → 停下来分析
- 卡住超过 8 轮循环 → 写 blocker 报告，等用户介入
- 不要在循环里改同一个地方期望不同的结果

## 完工标准
- [ ] 所有计划的验证步骤都通过了
- [ ] 每个验证都有可查证的证据
- [ ] 没有遗留的 TODO 或占位符
- [ ] 代码可以构建/运行
- [ ] 测试确实执行了（不是"理论能过"）
