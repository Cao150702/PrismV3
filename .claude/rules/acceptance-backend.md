# 后端验收标准 (Backend Acceptance)

## 测试分类

### 正常路径 (Happy Path)
- [ ] 正确的输入 → 正确的输出
- [ ] 标准的业务流程走通

### 异常路径 (Error Path)
- [ ] 错误的输入 → 正确的错误码 + 错误提示
- [ ] 缺失参数 → 400
- [ ] 无权限 → 401/403
- [ ] 资源不存在 → 404
- [ ] 重复创建 → 409
- [ ] 服务内部错误 → 500（但要有日志）

### 边界条件 (Edge Cases)
- [ ] 空字符串、空数组、null 值处理
- [ ] 超长输入处理
- [ ] 并发请求的安全性
- [ ] 超时场景的处理

### AI 接口专项
- [ ] 设计刁钻的测试场景，不只跑"最简单的情况"
- [ ] 测试模型返回异常格式时的容错
- [ ] 测试流式输出中断时的恢复

## 验证方式
- 必须贴出实际的 HTTP 响应（状态码 + body）
- 数据库中的数据确实变更了
- 日志中有对应的记录
